# Инструкция: Telegram-бот pre-boarding анкеты

## 1. Назначение

Бот последовательно задаёт вопросы из файла конфигурации. Пользователь отвечает в чате; ответы проверяются по правилам. После последнего вопроса все ответы сохраняются в отдельный файл.

**Команды в Telegram**

- `/start` — начать анкету заново (если опрос уже шёл, прогресс сбросится).
- `/cancel` — прервать анкету без сохранения результата.

---

## 2. Запуск бота на компьютере

1. Установите Python 3.10 или новее.
2. Откройте папку проекта `Onboarding Chat bot` в терминале.
3. Установите зависимости:

   `pip install -r requirements.txt`

4. Скопируйте файл `.env.example` в `.env`.
5. В `.env` укажите токен бота от [@BotFather](https://t.me/BotFather):

   `BOT_TOKEN=ваш_токен`

6. Запустите бота из корня проекта:

   `python -m bot.main`

Пока окно терминала открыто, бот онлайн. Для постоянной работы используйте свой сервер, планировщик задач или службу Windows.

**Не публикуйте** файл `.env` и токен в открытом доступе.

## 3. Где менять вопросы анкеты (не «ответы бота», а сценарий)

Вопросы и правила проверки задаются в файле **`data/survey.json`** (если не задан другой путь).

**Необязательно:** в `.env` можно указать другой файл:

`SURVEY_PATH=путь/к/вашему_опросу.json`

### Структура `survey.json`

- `title` — заголовок (можно показать в первом сообщении).
- `welcome` — приветственный текст перед первым вопросом.
- `done_message` — сообщение после успешного завершения.
- `questions` — массив вопросов. У каждого вопроса:
  - `id` — уникальный идентификатор (латиница, без пробелов, короткий).
  - `text` — текст вопроса для пользователя.
  - `type` — тип и валидация (см. ниже).
  - дополнительные поля зависят от типа.

### Типы вопросов (`type`)

- **`text`** — свободный текст. Поля: `min_length`, `max_length`, опционально `regex`.
- **`number`** — число. Поля: `min`, `max`, опционально `integer_only`: true.
- **`date`** — дата. Поле: `format` (Python), например `%d.%m.%Y`.
- **`email`** — почта. Опционально `pattern`.
- **`phone`** — телефон. Поля: `min_digits`, `max_digits` (по умолчанию 10 и 15).
- **`choice`** — кнопки. Поле: `choices` — массив подписей, например `["Да","Нет"]`.

После правок `survey.json` **перезапустите** бота (`Ctrl+C` и снова `python -m bot.main`), чтобы подтянуть изменения.

### Импорт вопросов из Word (.docx)

Каждый непустой абзац документа станет вопросом с типом `text`. Затем отредактируйте JSON (типы, кнопки, даты).

Пример (из папки проекта):

`python scripts/import_docx.py "C:\путь\к\файлу.docx" -o data/survey_imported.json`

Дальше либо переименуйте/скопируйте файл в `data/survey.json`, либо задайте `SURVEY_PATH` в `.env`.

## 4. Где брать результаты заполненных анкет

Завершённые анкеты дописываются в файл **`data/submissions.jsonl`** (по одной строке JSON на одну анкету), если не задан другой путь.

**Необязательно:** в `.env`:

`SUBMISSIONS_PATH=путь/к/файлу.jsonl`

### Что внутри каждой строки

- `submission_id` — уникальный номер записи.
- `telegram_user_id`, `username`, `first_name`, `last_name` — данные пользователя Telegram.
- `completed_at` — время завершения (UTC, формат ISO).
- `answers` — массив объектов с полями:
  - `question_id` — как в `survey.json`;
  - `question_text` — текст вопроса на момент прохождения;
  - `answer` — ответ пользователя.

### Как открыть

- Любой текстовый редактор или VS Code/Cursor — для просмотра и поиска.
- Excel: «Данные» → «Из текстового/CSV файла» (для одной строки удобнее сначала конвертировать JSONL в таблицу отдельной утилитой или скриптом).
- Для массового анализа можно написать короткий скрипт на Python, читающий файл построчно и собирающий таблицу.

Файл появляется после **первого успешного** завершения анкеты пользователем.

---

## 5. Краткий чек-лист

1. Настроить `.env` с `BOT_TOKEN`.
2. Отредактировать `data/survey.json` (или импорт из .docx + правка).
3. Запустить `python -m bot.main`.
4. В Telegram: `/start` → ответить на вопросы.
5. Результаты смотреть в `data/submissions.jsonl`.
